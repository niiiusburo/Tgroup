# TGroup Clinic — Glossary

> Every domain term, acronym, and concept with exact definitions. Disambiguations included for similar-sounding terms.

## A

**AccountPayment**  
Legacy read-only view or fallback table for historical payment records migrated from TDental. New code writes to `dbo.payments` and `dbo.payment_allocations`.

**ADR (Architectural Decision Record)**  
Append-only document capturing a context, decision, consequences, and rejected alternatives. Lives in `docs/ADR/ADR-NNNN.md`. Superseded by a newer ADR, never rewritten.

**Allocation**  
A row in `dbo.payment_allocations` linking a single `payment_id` to an `invoice_id` (saleorder) or `dotkham_id` with an `allocated_amount`. The canonical split-ledger for payments.

**AP (Appointment Name)**  
Auto-generated appointment identifier in the pattern `AP<6-digit-sequence>` (e.g., `AP000001`). Not user-editable. Generated by `api/src/routes/appointments.js` via `MAX(CAST(SUBSTRING(name FROM 3) AS INTEGER))`.

**API v1 vs API v2 (Enterprise)**  
- v1: Flat Express routes in `api/src/routes/*.js` performing direct SQL.  
- v2 (Enterprise): Clean architecture under `api/src/domains/` with `routes → controller → service → repository`.  
Both mount side-by-side in `api/src/server.js`. v2 routes are prefixed by their domain but do not use a global `/v2` prefix.

## B

**Balance (Customer)**  
Computed value: `SUM(payments.amount)` − `SUM(payment_allocations.allocated_amount)` for a given customer. Not stored; calculated on demand by `api/src/routes/customerBalance.js`.

**Blast Radius**  
The set of downstream files, routes, UI surfaces, and tests that may break when a given file or schema entity changes. Documented in `product-map/contracts/dependency-rules.yaml` and `DEPENDENCY-MAP.md`.

## C

**Check-In Flow**  
The operational sequence: patient arrives → status transitions `not-arrived → arrived → waiting → in-treatment → done`. Managed in `AppointmentHoverContext` and `PatientCheckIn` components. Face recognition is an optional accelerator, not a requirement.

**Color (Appointment)**  
A string code (`'0'` through `'7'`) stored on `dbo.appointments.color` and mapped to hex values in `website/src/constants/index.ts` (`APPOINTMENT_CARD_COLORS`). Hardcoded on both frontend and backend.

**Commission**  
Staff payout calculated from `commissionproductrules` and stored in `saleorderlinepartnercommissions`. The auto-calculation trigger is currently unknown (see `product-map/unknowns.md` #12).

**Compreface**  
External face-recognition service (Exadel, Java-based) run as Docker containers. Legacy fallback; primary face recognition uses the local Python `face-service` (OpenCV YuNet + SFace).

**Company**  
A clinic location/branch. Table `dbo.companies`. Referenced as `companyid` or `company_id` across nearly every entity.

**Contract (Shared)**  
TypeScript/Zod schema in `contracts/*.ts`, built into `@tgroup/contracts` and consumed by both frontend and backend. Examples: `AppointmentCreateSchema`, `PaymentCreateSchema`.

**CSKH**  
Vietnamese abbreviation for Customer Service / Customer Care (`Chăm sóc khách hàng`). In the codebase, `cskhid` is a column on `dbo.partners` linking a customer to their assigned CSKH employee.

## D

**Deposit**  
A prepayment (`payment_category = 'deposit'`, `deposit_type = 'deposit'`) that can be allocated to outstanding invoices or held as wallet balance. Receipt numbers follow `TUKH/YYYY/NNNNN`.

**Deposit Used**  
The portion of a payment that draws from a customer's existing deposit wallet (`payments.deposit_used`). Tracked separately from `cash_amount` and `bank_amount`.

**Dotkham (Đợt khám)**  
A medical treatment session/record. Table `dbo.dotkhams` links to `dbo.dotkhamsteps` for procedure details. Read-only in current UI; writes may happen via Odoo sync or legacy TDental import.

## E

**Effective Permissions**  
The resolved permission set for an employee at runtime: `group_permissions` (via `partners.tier_id`) + `permission_overrides` − location scope restrictions. Computed by `resolveEffectivePermissions()` in `api/src/services/permissionService.js`.

**Embedding (Face)**  
A 128-dimensional vector produced by the SFace model. Stored in the `face-service` SQLite or `dbo.customer_face_embeddings` (status unclear; see `product-map/unknowns.md`).

**e-Invoice**  
Electronic invoice fields on `dbo.partners`: `unitname`, `unitaddress`, `taxcode`, `personalname`, `personalidentitycard`, `personaltaxcode`, `personaladdress`. Downstream consumer unknown (see `product-map/unknowns.md` #5).

**Enterprise Route**  
See **API v1 vs API v2 (Enterprise)**.

## F

**Face Subject ID**  
The unique identifier assigned by the face-recognition engine (Compreface or face-service) to a registered face. Stored on `dbo.partners.face_subject_id`.

**FIFO Allocation**  
Default deposit allocation strategy: payments are applied to the oldest outstanding invoice first. Implemented in frontend `allocatePaymentSources.ts` and backend `payments.js` helpers.

## G

**Group Permission**  
A row in `dbo.group_permissions` mapping a `permission_groups.id` to a permission string (e.g., `customers.view`). The canonical permission registry.

## H

**Hard Delete**  
Permanent row removal (`DELETE FROM ...`). Contrasts with **Soft Delete**. For `dbo.partners`, hard delete is available at `DELETE /api/Partners/:id/hard-delete` and requires `customers.hard_delete` permission.

**Hosoonline**  
External health-checkup image provider. TGClinic proxies image reads because direct URLs require Hosoonline session auth. Uses `HOSOONLINE_BASE_URL`, `HOSOONLINE_USERNAME`, `HOSOONLINE_PASSWORD` env vars.

## I

**IP Access Control**  
Per-company IP whitelist feature. Tables `dbo.ip_access_settings` and `dbo.ip_access_entries`. Enforced by `api/src/middleware/ipAccess.js`.

**Invoice**  
In payment allocation context, an `invoice_id` refers to `dbo.saleorders.id`. In legacy TDental context, it may refer to an Odoo invoice record.

## J

**JWT**  
JSON Web Token used for session auth. Stored in `localStorage` as `tgclinic_token`. Expires in `24h` (default) or `60d` (remember me). Signed with `JWT_SECRET`.

## L

**Legacy Route**  
Deprecated API route kept for potential external clients. Examples: `/api/Account/Login` (`api/src/routes/account.js`), `/api/Web/Session` (`api/src/routes/session.js`). Do not delete without checking `product-map/unknowns.md`.

**Location Context**  
React context (`website/src/contexts/LocationContext.tsx`) filtering list views by the user's selected clinic branch. Applied frontend-side; backend does not enforce location scope on most list routes.

## M

**Method (Payment)**  
Enum on `dbo.payments.method`: `cash`, `bank_transfer`, `deposit`, `mixed`. Frontend type in `website/src/types/payment.ts`. Backend Zod schema in `contracts/payment.ts` also accepts `card`, `momo`, `vnpay`, `zalopay` but these may not be fully wired.

**Mixed Payment**  
A payment combining cash, bank transfer, and deposit wallet in a single transaction. Stored with `method = 'mixed'` and broken down in `cash_amount`, `bank_amount`, `deposit_used`.

**Monthly Plan**  
Installment payment plan for large treatments. Tables `dbo.monthlyplans`, `dbo.monthlyplan_items`, `dbo.planinstallments`. Payments can target a specific installment via `PUT /api/MonthlyPlans/:id/installments/:installmentId/pay`.

## N

**Normalize Text**  
Accent-insensitive string normalization required for all project search bars. Vietnamese names with diacritics must match when staff type without accents. Frontend uses `normalizeText()`; backend strips accents in SQL `WHERE` clauses.

## O

**Odoo**  
Legacy ERP system (predecessor to TGClinic). Many table names (`saleorders`, `saleorderlines`, `productcategories`, `stockPickings`) and column conventions (`categid`, `salestaffid`) are Odoo legacies.

## P

**Partner**  
Universal person table (`dbo.partners`) using single-table inheritance. A partner can be a customer (`customer=true`), employee (`employee=true`), supplier (`supplier=true`), doctor (`isdoctor=true`), assistant (`isassistant=true`), or receptionist (`isreceptionist=true`).

**Payment Category**  
Computed classification: `'payment'` (regular service payment) or `'deposit'` (prepayment/refund). Determined by heuristics in `api/src/routes/payments.js` when `deposit_type` is absent.

**Permission Override**  
A row in `dbo.permission_overrides` granting or revoking a specific permission string for an individual employee, bypassing their tier group defaults.

**Primary Branch**  
An employee's default location from `partners.companyid`. Combined with `employee_location_scope` rows to produce the full location access list at login.

**Product**  
In TGClinic, a **product** is a dental **service** (type = `'service'` in the legacy schema). Table `dbo.products`. Do not confuse with physical retail products.

## R

**Receipt Number**  
Auto-generated receipt for deposits: `TUKH/YYYY/NNNNN` (incrementing per calendar year). Generated by `generateReceiptNumber()` in `api/src/routes/payments/helpers.js`.

**Ref**  
Customer code stored on `dbo.partners.ref`. Migrated from TDental. Not guaranteed unique; used for operational search and display.

**Remember Me**  
Login checkbox extending JWT expiry from `24h` to `60d`. Stored in token payload as `remember: true`.

**Residual**  
Remaining balance on a saleorder invoice: `saleorders.residual`. Updated by payment allocations. Must never go negative (enforced by `GREATEST(0, residual - amount)` and pre-insert validation).

## S

**SaleOrder**  
Treatment plan / invoice. Table `dbo.saleorders`. State machine: `draft → confirmed → done → cancelled`. Commission and payment allocation targets.

**Schema Map**  
`product-map/schema-map.md` — documents every table's writers, readers, endpoints, UI surfaces, and risk level.

**SMI (Single-Table Inheritance)**  
Database pattern where multiple logical entities (customers, employees, suppliers) share one physical table (`dbo.partners`) with boolean discriminator columns.

**Soft Delete**  
Row marked `isdeleted = true` without physical removal. Used for `dbo.partners` and several other tables. Contrasts with **Hard Delete**.

## T

**TDental**  
Predecessor clinic management system. Data imported via CSV scripts in `api/scripts/tdental-import/`. Many tables retain TDental column names and data anomalies.

**Telemetry**  
Error event ingestion system. Tables `dbo.error_events` and `dbo.error_fix_attempts`. Public endpoint `POST /api/telemetry/errors` (no auth required) for frontend error reporting.

**Tier**  
Permission group assigned to an employee via `partners.tier_id` → `permission_groups.id`. Replaced hard-coded role checks. See **Effective Permissions**.

**Time Expected**  
Appointment duration in minutes (`dbo.appointments.timeexpected`). Valid range `[1, 480]`. Default 30.

**Tooth Range**  
Serialized JSON array of tooth numbers stored on `dbo.saleorderlines.tooth_numbers`. Used by the tooth picker UI during service entry.

## U

**Unknowns**  
Unresolved questions or intentionally unimplemented areas tracked in `product-map/unknowns.md`. Agents must stop and clarify before implementing work that intersects an unknown.

## V

**VietQR**  
QR code generation for Vietnamese bank transfers. Payload built from `company_bank_settings.vietqr_payload` or generated via `website/src/lib/vietqr.ts`.

## W

**Wallet (Deposit)**  
Customer's available deposit balance. Computed as `SUM(deposits) - SUM(deposit usage)` from `dbo.payments` and `dbo.payment_allocations`. Displayed in `DepositWallet` component.

**Worktree**  
Git worktree used for parallel feature/fix lanes. See `AGENTS.md` §12 and `.agents/skills/wtree/SKILL.md`. Big features and small fixes must use separate worktrees.

## Similar-Sounding Terms — Disambiguation

| Term A | Term B | How to Tell Them Apart |
|---|---|---|
| **Payment** (canonical money row) | **AccountPayment** (legacy read fallback) | Payment = `dbo.payments` (new). AccountPayment = legacy view/table. |
| **Deposit** (prepayment category) | **Deposit Used** (wallet draw on a payment) | Deposit = the payment row itself. Deposit Used = a field on a later payment row. |
| **Partner** (any person) | **Customer** (`customer=true` partner) | Partner = table. Customer = role within that table. |
| **Employee** (`employee=true` partner) | **Doctor** (`isdoctor=true` partner) | Employee = anyone on payroll. Doctor = a clinical role; a doctor is also an employee. |
| **Service** (user-facing term) | **Product** (legacy table name) | They are the same thing. Table = `products`. UI label = "Dịch vụ" / Service. |
| **Invoice** (payment allocation target) | **SaleOrder** (treatment plan record) | In allocations, `invoice_id` = `saleorders.id`. Conceptually, a SaleOrder becomes an invoice when it has a residual > 0. |
| **Company** (table name) | **Location** (user-facing term) | They are the same thing. Table = `companies`. UI label = "Chi nhánh" / Location. |
| **Face Subject ID** (engine identifier) | **Partner ID** (database UUID) | Face Subject ID is assigned by Compreface/face-service. Partner ID is the DB PK. |
| **Allocation** (payment split row) | **Payment** (money row) | One payment can have many allocations. |
| **v1 Route** (flat Express) | **v2 / Enterprise Route** (clean arch) | v1 = `api/src/routes/*.js`. v2 = `api/src/domains/*/routes.js`. |
