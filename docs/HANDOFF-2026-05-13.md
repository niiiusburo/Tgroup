### 2. Permission Matrix Fix (Complete)

**Root Cause:** The permission matrix UI updated React state locally but never called backend APIs. Every toggle, add, remove, and location scope change was a no-op.

**Solution:** Full refactor (Approach C) — split god-hook into two focused hooks, wire all mutations to real APIs.

#### Backend Changes

| Endpoint | Change |
|---|---|
| `DELETE /api/Permissions/groups/:id` | **New** — deletes group, unassigns members, rejects system groups |
| `PUT /api/Permissions/employees/:id` | **Enhanced** — supports `groupId: null` for unassignment |

#### Frontend Changes

| File | Change |
|---|---|
| `website/src/hooks/usePermissionGroups.ts` | Rewritten — group CRUD only, real API calls, optimistic updates |
| `website/src/hooks/useGroupMembers.ts` | **New** — member assignment/locations/overrides management |
| `website/src/components/settings/PermissionGroupConfig.tsx` | Uses both hooks, loading spinners, error banners |
| `website/src/lib/api/permissions.ts` | Added `deletePermissionGroup`, `unassignEmployeePermission` |
