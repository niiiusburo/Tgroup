# ADR-0003: Local-First Release Discipline

## Status
Accepted

## Context
Production deploy issues have happened when code shipped before migrations, permissions, nginx timeouts, or local verification were complete. Agents were occasionally patching VPS files directly as a first fix path.

## Decision
All fixes are made and verified locally before VPS changes. VPS deployment happens only after local code, data, migration, and behavior checks are complete. Agents must not patch VPS files directly as the first fix path.

## Consequences
- Local dev environment must be kept functional; if it breaks, fix it before shipping.
- Migrations are applied locally first, then manually on the VPS after deploy (migrations are not auto-run).
- Every deploy recap must include local verification evidence.
- Slower emergency response but higher safety.

## Alternatives Rejected
- **Direct VPS hotfixing:** Faster in the moment, but untested changes have caused 500s, permission lockouts, and data corruption in production.
- **Auto-deploy on merge:** Rejected because migrations are manual and must be timed with code deploys.
