# ADR-0005: Manual SQL Migrations (No ORM Runner)

## Status
Accepted

## Context
The project uses raw `pg` queries and has 54+ SQL migration files in `api/migrations/`. There is no Knex, Sequelize, node-pg-migrate, or similar runner.

## Decision
Continue with manual SQL migrations. Each migration file is idempotent (uses `IF NOT EXISTS` / `IF EXISTS` where possible). Migrations are tracked in a simple `schema_migrations` table (`000_install_schema_migrations_table.sql`). They are applied manually via `docker exec -i tgroup-db psql ... < file.sql` or equivalent.

## Consequences
- No migration framework dependency; full control over SQL.
- Risk of missed migrations on deploy (human must remember to run them).
- Schema changes require writing SQL by hand; no auto-generated DDL.
- Every migration must be idempotent because the same file may be run twice by accident.

## Alternatives Rejected
- **Adopt node-pg-migrate:** Would add a dependency and require rewriting all existing migrations. The team preferred keeping raw SQL for complex PostgreSQL-specific features.
- **Use an ORM (Sequelize/Prisma):** Would hide the legacy schema complexity and make Odoo-legacy column names harder to manage.
