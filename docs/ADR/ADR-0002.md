# ADR-0002: Product Map Remains Mandatory

## Status
Accepted

## Context
`product-map/` already tracks domain ownership, schema blast radius, dependency rules, permissions, unknowns, hooks, and tests. Some agents suggested replacing it with a simpler README-based approach.

## Decision
Do not replace `product-map/` with new root docs. Root docs route work; `product-map/` governs domain-specific implementation details. Any schema, API, permission, or UI change must still read the affected domain YAML, `schema-map.md`, `contracts/dependency-rules.yaml`, and `unknowns.md`.

## Consequences
- Domain YAMLs (`product-map/domains/*.yaml`) remain the mandatory first read for feature/bugfix work.
- Schema blast radius stays explicitly documented in `product-map/schema-map.md`.
- Change-type checklists in `product-map/contracts/dependency-rules.yaml` must be followed.
- Keeping the map alive is a shared responsibility: if code drifts from the product-map, update the relevant artifact.

## Alternatives Rejected
- **Merge product-map into root docs:** Would create a single oversized document and lose the per-domain ownership model.
- **Delete product-map and rely on code comments:** Comments drift; the YAML format is structured enough for agents to read programmatically.
