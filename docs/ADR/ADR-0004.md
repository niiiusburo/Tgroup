# ADR-0004: Single JWT in localStorage (Not httpOnly Cookie)

## Status
Accepted

## Context
TGClinic stores the JWT token in `localStorage` under `tgclinic_token`. Security reviews sometimes suggest moving to httpOnly cookies to prevent XSS token theft.

## Decision
Keep JWT in `localStorage`. The app runs as a SPA with CORS to the API; cookie-based sessions would require CSRF protection and complicate the mobile/tablet usage patterns observed in clinics. Instead, mitigate XSS via:
- Helmet CSP headers.
- Input sanitization on all user-generated content.
- Short token lifetime (24h default, 60d max with remember me).
- No sensitive actions without re-authentication for destructive operations (future hardening).

## Consequences
- XSS vulnerability surface exists if DOM is compromised.
- Simpler SPA auth flow; no CSRF token management needed.
- Token refresh/re-auth is manual (login page).

## Alternatives Rejected
- **httpOnly cookie + refresh token:** Adds complexity; no mobile app currently benefits from cookie auth. Revisit if a native mobile app is built.
- **OAuth / SSO:** Overkill for current clinic staff model; all users are internal employees.
