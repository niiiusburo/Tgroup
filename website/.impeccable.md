# TGClinic Design Context

> This file provides the design context required by the `frontend-design` and `teach-impeccable` skills.
> Do not infer this from the codebase — this is the canonical source of truth.

---

## Target Audience

Dental clinic administrators, receptionists, and dentists who operate the software daily for 4–8 hours. They are not technical users. They need speed, clarity, and reliability.

## Use Cases

- Morning check-in: View today's appointments, check in patients, handle walk-ins
- Scheduling: Book, reschedule, cancel appointments across multiple doctors and rooms
- Patient management: Register new patients, update records, view visit history
- Billing: Process payments, view monthly plans, track revenue
- Reporting: Generate daily/weekly/monthly performance reports
- Staff management: Manage employee schedules, permissions, commissions

## Brand Personality

**Clean, professional, trustworthy, warm.**

This is medical software, but it should not feel cold or sterile. The warm orange primary color conveys energy and human care. The interface should feel organized and predictable — users should never wonder where something is or what will happen when they click.

## Tone

Confident but approachable. Interfaces should feel like a capable assistant, not a commanding boss. Error messages should be helpful, not alarming. Success states should be subtle, not celebratory.

## Differentiation

Unlike generic clinic software that looks like a spreadsheet, TGClinic feels modern and intentional. The calendar is the heart of the experience — everything else radiates from it. The design prioritizes information density without clutter.

## Constraints

- **Framework**: React 18 + Vite + Tailwind CSS
- **Performance**: Must work on mid-range tablets used at reception desks
- **Accessibility**: WCAG 2.1 AA minimum — dental staff may have varying vision abilities
- **i18n**: Must support English and Vietnamese (RTL not required)
- **Responsive**: Must be usable on 13" laptops (primary) and iPad Mini (secondary)

## Visual Direction

- **Light mode only** — the app runs in bright clinic environments
- **Warm neutrals** — tinted toward orange, never cold gray
- **Rounded but not bubbly** — 10px–16px radius feels friendly but professional
- **Shadows for elevation** — cards lift subtly, modals lift more
- **Motion is functional** — transitions guide attention, never entertain
