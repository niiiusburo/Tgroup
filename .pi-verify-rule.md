# Pi Verification Rule: HTML Preview Verification (Playwright)

**Applies to:** All HTML preview files generated for UI/layout review before code changes.

## Rule
Before delivering any HTML preview URL or file path to the user, I MUST:

1. Run an automated Playwright verification against the generated HTML file.
2. Confirm the preview renders without errors (screenshot must succeed).
3. Assert visibility of the key UI sections mentioned by the user (e.g., left cards, tabs, footer buttons).
4. Only report the preview as "ready" after all assertions pass.

## Verification Checklist
- [ ] Screenshot captured successfully via Playwright
- [ ] Key page sections are visible and populated
- [ ] No 404 / net::ERR_EMPTY_RESPONSE / console errors blocking render
- [ ] If serving via localhost, prefer `file://` protocol to avoid server flakiness

## Automation Script (Python + Playwright)
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch()
    page = browser.new_page(viewport={"width": 1400, "height": 900})
    page.goto("file:///absolute/path/to/preview.html")
    page.wait_for_timeout(1000)
    page.screenshot(path="/absolute/path/to/preview-verification.png", full_page=True)
    # Assert key elements
    assert page.locator("text=KEY_SECTION_1").is_visible()
    # ... more asserts
    browser.close()
```
