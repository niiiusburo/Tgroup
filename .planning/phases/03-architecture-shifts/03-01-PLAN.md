    Modify api/src/routes/employees.js:

    1. GET /api/Employees list:
       - After fetching items, for each employee collect location scopes:
         const scopeRows = await query('SELECT company_id FROM employee_location_scope WHERE employee_id = ANY($1)', [items.map(i => i.id)]);
       - Attach locationScopeIds to each item:
         item.locationScopeIds = scopeRows.filter(r => r.employee_id === item.id).map(r => r.company_id);

    2. GET /api/Employees/:id:
       - After fetching employee, also query:
         const scopes = await query('SELECT company_id FROM employee_location_scope WHERE employee_id = $1', [id]);
       - Attach locationScopeIds: rows[0].locationScopeIds = scopes.map(s => s.company_id);

    3. POST /api/Employees:
       - Accept locationScopeIds: string[] in req.body (optional, default []).
       - Wrap partner INSERT + scope INSERTs in a transaction using query('BEGIN'), query('COMMIT'), query('ROLLBACK') on error.
       - After creating employee (insert into partners), insert into employee_location_scope for each scope company_id that is NOT the primary companyid (deduplicate).
       - Return the created employee with locationScopeIds.

    4. PUT /api/Employees/:id:
       - Accept locationScopeIds: string[] in req.body (optional).
       - Begin transaction.
       - Run existing UPDATE partners query.
       - If locationScopeIds is provided:
         DELETE FROM employee_location_scope WHERE employee_id = $1;
         Then INSERT new scope records for each company_id (skip primary companyid to avoid duplicate, but enforce at DB layer).
       - Commit. Return updated employee with locationScopeIds.

    Use parameterized queries throughout. Handle errors with rollback.
  </action>
